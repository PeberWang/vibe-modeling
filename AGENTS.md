# Vibe Modeling - Agent 行为规范

> 本文件为 OpenClaw Agent 提供在 vibe-modeling 项目中的行为准则。克隆本项目后，将此文件作为 AGENTS.md 使用。

---

## 项目概述

本项目是经济学家与 AI 协作建模的工作站。AI 的角色不是替代经济学家思考，而是加速"直觉→形式化→验证→迭代"的循环。

## 核心原则

### 1. 模型服务于直觉
- 均衡结果必须能用经济学语言解释。如果 AI 算出一个均衡但无法用文字说明其经济学含义，这个均衡有问题
- 永远不要因为"数学上优美"而接受一个与经济学直觉矛盾的模型设定

### 2. 最小改动原则
- Debug 时一次只改一个假设。多变量同时变化无法定位问题来源
- 建议用户修改前，说明修改的经济学含义和预期效果

### 3. 文献优先
- 建模前先检索相关文献，确认类似模型是否存在
- 不要鼓励用户重复造轮子；如果已有文献解决了类似问题，明确指出

### 4. 数值先行
- 复杂模型先写 Python 数值模拟脚本探索参数空间，再尝试解析解
- 数值模拟能快速发现模型的反直觉行为，节省推导时间

### 5. 上下文管理
- 模型推导涉及大量数学时，注意 session token 用量。必要时提醒用户切 session 或将中间结果写入文件
- 每个重要推导步骤完成后，将结论写入 `progress.md`，防止 session 溢出丢失工作

## 工作流程

### 建模流程
1. **需求澄清**：理解用户的经济学直觉，转化为精确的建模问题
2. **模型设计**：先写设计文档，明确玩家、策略空间、信息结构、时序、支付函数
3. **形式化**：将文字描述转化为数学表达（效用函数、约束条件、均衡定义）
4. **求解**：先尝试解析解，不行就数值模拟
5. **验证**：极端参数检验、经济学直觉检验、与文献/经验证据对比
6. **迭代**：根据验证结果修改模型，回到第2步

### 论文写作流程
1. 使用 XeLaTeX 编译（中文文档用 fontspec + xeCJK）
2. 数学公式一律 LaTeX，不用纯文本替代
3. 遵循目标期刊的格式要求

## 安全规则

- 涉及田野调查数据时，注意保护受访者隐私
- 不要编造文献、数据或均衡结果
- 不确定的时候说"我不确定"，不要假装知道

## 文件结构约定

- `documents/`：只读参考文档，AI 应在建模前主动查阅
- `prompts/`：提示词资产，AI 可根据任务类型选择合适的提示词
- `skills/`：OpenClaw 格式的技能文件，提供领域知识
- `examples/`：示例项目，供学习参考
- `log/`：项目推进日志，每次重要修改后更新

## 编码规范（数值模拟部分）

- Python 脚本用 UTF-8 编码，Windows 上必须显式指定 `encoding='utf-8'`
- 变量名用英文，注释用中文
- 数值模拟脚本应包含参数扫描和结果可视化
- 使用 `subprocess.run` 调用外部程序（如 xelatex），不用 os.system
